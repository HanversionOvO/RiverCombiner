## Description

认证模块为 river 提供双通道登录体系，支持 RiverSide 原生账号密码登录和清水河畔 OAuth/WebView 授权登录两种模式。登录入口通过 `LoginPage` 统一展示，用户可选择不同的认证提供方完成身份验证。

核心状态管理由 `AccountStore`（ChangeNotifier）驱动，负责维护当前登录账号、Provider 选择以及会话生命周期。`AccountModels` 定义了 `AccountProvider`、`UserAccount`、`QingShuiHePanAuth` 等关键数据结构，区分不同登录方式的认证凭证和用户信息格式。`RiverSideSessionReader` 负责从持久化存储中恢复会话状态，确保应用重启后用户无需重复登录。

RiverSide 密码登录由 `RiverSidePasswordLoginService` 处理标准的用户名/密码认证流程，返回 session token。清水河畔登录则通过 `QingShuiHePanLoginService` 发起 OAuth 授权，`RiverSideLoginWebViewPage` 承载 WebView 交互页面，支持完整的 OAuth 回调处理和外部浏览器降级回退。`RiverSideAccountSettingsModels` 提供账号设置相关的数据模型（如邮箱状态、称号徽章、认证令牌），与 FT-008 用户设置模块共享。

架构上采用 Service-Page-Store 三层分离：Service 层封装 API 调用和认证协议细节，Page 层负责 UI 呈现和用户交互，Store 层管理全局认证状态的响应式分发。
