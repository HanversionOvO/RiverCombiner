## Description

Core Infrastructure forms the foundational layer that all other features of the river platform depend upon. At its center is `RiverSideApiClient`, a centralized HTTP client that organizes all server communication through extension-based endpoint grouping -- topics, posts, search, reactions, notifications, chat, profile, account settings, and message bus each have their own client extension, keeping the surface area manageable while sharing a common auth, error-handling, and serialization pipeline. Cookie-based authentication is bridged through the platform layer, ensuring session continuity between the WebView and native contexts.

Supporting services fill out the infrastructure. `RiverServerConfig` provides runtime server endpoint resolution, allowing the app to connect to different environments (development, staging, production) without code changes. `AppCacheService` implements an LRU-based cache management strategy, with configurable size limits and time-to-live policies, serving as the caching backbone for image assets, API responses, and mini-app resources. `PlatformBridges` abstracts platform-specific capabilities -- WebView configuration, custom font loading, and dynamic app icon switching -- behind a unified Dart interface, ensuring the Flutter layer remains portable.

The shared UI and utility layer includes `SharedWidgets`, a library of reusable components (snackbar, confirmation dialog, emoji picker, AI action button, home widget service) that enforce visual consistency across features. `RiverAiService` wraps AI provider integration with streaming response support, abstracting provider selection and request formatting behind a single service interface. `PicUiImageHostService` integrates with the PicUI image hosting backend for upload, transform, and delivery. `AppUpdateChecker` polls for new app versions and surfaces update prompts. `QingEmojiCatalog` provides the local emoji asset catalog used throughout the app, while `Constants` and `Categories` supply centralized configuration values and content taxonomy definitions.
