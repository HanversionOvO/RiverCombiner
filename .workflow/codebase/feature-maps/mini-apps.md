## Description

The Mini Apps feature implements a complete lightweight application ecosystem within the river platform, enabling third-party developers to publish and users to discover, install, and run mini-applications. Each mini-app is defined by an app manifest (`RiverMiniAppModels`) that captures versioning metadata, permission declarations, entry-point configuration, and asset references. `RiverMiniAppRepository` serves as the discovery and catalog layer, handling app listing, search, and submission flows through the platform client.

Installation and lifecycle management are distributed across a set of purpose-built stores. `RiverMiniAppInstallStore` manages the install/uninstall lifecycle, tracking installed apps and handling storage allocation and cleanup. `RiverMiniAppHostStore` provides the WebView-based hosting runtime that sandboxes each mini-app's execution, managing navigation, script injection, and bridge communication with the host platform. `RiverMiniAppFloatingStore` implements the floating dock UI pattern -- a persistent, collapsible overlay that allows users to open and switch between mini-apps without leaving their current context, complete with minimize, maximize, and dismiss gestures.

Security and sharing round out the ecosystem. `RiverMiniAppPermissionStore` enforces a granular permission model with policy management, prompting users for consent when a mini-app requests access to platform capabilities (camera, location, user data) and maintaining per-app permission state across sessions. `RiverMiniAppCodecs` provides barcode and image codec utilities that encode app identifiers into shareable QR codes and decode scanned codes for rapid app discovery and installation. `MiniAppPages` presents the user-facing UI -- app catalog, detail views, permission dialogs, and the submission flow for developers uploading new apps.
