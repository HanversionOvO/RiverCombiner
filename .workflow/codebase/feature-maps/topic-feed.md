## Description

话题流模块是 river 的核心内容消费入口，以 `PostsPage` 为展示载体，提供最新帖子和热门帖子两种排序方式。用户可在最新/热门之间切换，页面基于滚动分页加载，通过 `RiverSideApiClientPosts` 发起 API 请求获取分页数据。

内容模型层由 `RiverSideTopicModels` 定义，包含 `RiverSideTopicFeed`（流式列表）、`RiverSideTopicSummary`（帖子摘要）、`RiverSideTopicPage`（分页结构）等核心数据结构，同时复用于 FT-004 话题详情模块。`RiverSideApiClientTopics` 提供话题相关的 API 方法扩展，与 Posts API 协同完成完整的话题数据链路。

足迹追踪由 `TopicFootprintStore` 实现，记录用户已浏览的帖子列表（`TopicFootprintEntry`），在帖子列表中可视化区分已读/未读状态。该 Store 基于本地持久化，跨会话保留用户的阅读历史，同时提供清理和同步机制。整体架构遵循 RiverSide API 的通用分页协议，数据流从 API Client 经 Model 反序列化后驱动 Page 层 Widget 重建。
