## Description

The Notifications & Real-time feature provides the event-driven communication backbone of the river platform, combining a persistent notification feed with real-time message delivery. The `NotificationsPage` presents a unified inbox that categorizes incoming events into channel notifications (forum replies, mentions, system announcements) and direct messages from other users. Each notification row surfaces key metadata -- sender avatar, content preview, timestamp, and read/unread state -- with tap-to-navigate behavior that deep-links into the relevant conversation or post.

Real-time capability is driven by a layered polling architecture. `RiverSideMessageBusPoller` maintains a lightweight long-polling connection to the server-side message bus, receiving push-style events without requiring WebSocket infrastructure. The poller feeds into `RiverSideRealtimeInboxService`, which aggregates events across sessions, deduplicates messages, and dispatches in-app banner notifications that float over the current screen when the user is outside the notifications page. `RiverSideNotificationModels` provides the typed event schemas -- distinguishing channel messages, direct messages, reaction events, and system notices -- enabling the UI layer to render each notification type with appropriate visual treatment.

The `ChatDetailPage` builds on this infrastructure to provide a full chat experience for direct message threads, including message bubbles, typing indicators, and reaction support (emoji quick-reply on individual messages). The page maintains its own scroll-state and read-watermark logic, syncing back to the inbox service so that the notification badge count accurately reflects unread state across the app.
