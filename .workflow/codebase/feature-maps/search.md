## Description

The Search feature provides a unified search experience across the river platform, allowing users to discover content spanning posts, users, and mini-apps from a single entry point. The search interface uses a tabbed result layout that segments hits by content type, giving users immediate visual separation between matching discussions, member profiles, and available mini-applications. Each result card highlights matched text fragments inline, making it immediately obvious why a given result surfaced for the user's query.

Under the hood, `RiverSideApiClientSearch` extends the centralized API client with search-specific endpoints, handling query construction, result deserialization, and pagination through cursor-based offsets. `RiverSideSearchModels` provides the typed data structures that bridge raw API responses into the UI layer, including models for search suggestions, result items with highlight spans, and filter state. The `SearchPage` orchestrates the full search flow -- from debounced text input through API dispatch to staggered list animations -- and supports forum-specific filtering so users can scope searches to particular discussion boards.

The architecture follows the established platform pattern of separating concerns across client, model, and page layers. Network calls flow through the API client extension, which handles auth cookies and error normalization, while the models layer provides immutable data classes consumed by the page's state management. Staggered animations on result appearance contribute to a polished, responsive feel without blocking the critical path of search result delivery.
